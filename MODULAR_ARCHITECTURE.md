# Sontha Messenger - Модульная архитектура

## Структура проекта

Проект был разделен на модули для улучшения производительности и организации кода:

### 📁 Структура файлов

```
src/
├── js/
│   ├── core.js          # Основная логика и утилиты (~400 строк)
│   ├── auth.js          # Аутентификация и управление пользователями (~300 строк)
│   ├── chat.js          # Чаты и сообщения (~500 строк)
│   ├── call.js          # Голосовые и видеозвонки (~400 строк)
│   ├── ui.js            # Интерфейс, темы и меню (~500 строк)
│   └── features.js      # Дополнительные функции (~600 строк)
├── app.js               # Главный файл инициализации (~50 строк)
├── app.js.backup        # Резервная копия оригинального файла
├── index.html           # HTML с подключением модулей
└── styles.css           # Стили
```

### 🔧 Модули

#### 1. **core.js** - Основной модуль
- Socket соединение
- Глобальное состояние приложения
- Утилитарные функции
- Управление чатами
- Уведомления

#### 2. **auth.js** - Аутентификация
- Регистрация и вход
- Управление токенами
- Captcha система
- Выход из системы

#### 3. **chat.js** - Чаты и сообщения
- Отправка и получение сообщений
- Редактирование и удаление
- Реакции на сообщения
- История звонков

#### 4. **call.js** - Звонки
- Голосовые и видеозвонки
- WebRTC соединения
- Управление медиа
- История звонков

#### 5. **ui.js** - Интерфейс
- Система тем
- Меню и навигация
- Профиль пользователя
- Настройки
- Избранное

#### 6. **features.js** - Дополнительные функции
- Групповые чаты
- Голосовые сообщения
- Каналы
- Секретные чаты

### 🚀 Преимущества модульной архитектуры

1. **Производительность**
   - Ленивая загрузка модулей
   - Кэширование отдельных модулей
   - Параллельная загрузка

2. **Организация кода**
   - Четкое разделение ответственности
   - Легкость поддержки
   - Переиспользование кода

3. **Масштабируемость**
   - Простое добавление новых функций
   - Независимое тестирование модулей
   - Гибкая архитектура

4. **Отладка**
   - Изолированные ошибки
   - Простое отслеживание проблем
   - Четкие границы модулей

### 📦 Порядок загрузки

Модули загружаются в следующем порядке:

1. `core.js` - Основные функции
2. `auth.js` - Аутентификация
3. `chat.js` - Чаты
4. `call.js` - Звонки
5. `ui.js` - Интерфейс
6. `features.js` - Дополнительные функции
7. `app.js` - Инициализация

### 🔄 Совместимость

- Все функции работают как раньше
- API остался неизменным
- Обратная совместимость сохранена
- Оригинальный файл сохранен как `app.js.backup`

### 🛠️ Разработка

Для добавления новых функций:

1. Определите подходящий модуль
2. Добавьте функцию в соответствующий файл
3. Экспортируйте через `window.ModuleName`
4. Обновите документацию
